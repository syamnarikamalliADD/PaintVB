PLCR-005 OP STOP Teach Pendant Enabled In Manual or Auto

Cause:
Robot Teach Pendant was enabled when the system was in Auto or Manual mode. This event causes a one time fault to stop any motion. If the system is in auto mode the controller will abort all jobs being executed.

Remedy:
Disable the Teach Pendant and press fault reset to reset the fault.

PLCR-006 OP STOP In-Cycle Switched From Auto

Cause:
Booth was running in auto and was turned to manual or teach while in cycle.

Remedy:
Move robots to the Home Position, then place the system back in Auto Mode.

PLCR-007 OP STOP Teach Pendant E-Stop

Cause:
Teach Pendant E-Stop button Pressed.

Remedy:
Release the teach pendant E-STOP button to clear the E-STOP condition.

PLCR-008 OP STOP DCS Disabled or Signature Changed

Cause:
DCS settings have changed causing a DCS Signature Mismatch.

Remedy:
Go to DCS settings--> signature number, select Annum from the button menu and then select CNF SIG.

PLCR-019 SYS FAULT Robot Servo Enable SS- Cycle Inputs

Cause:
While Output 1 is active, channel A or channel B transitioned from the active state to the safe state and back to the active state before the other input channel transitioned to the safe state.

Remedy:
Cycle Inputs - Cycle Inputs prompts for action. Before Output 1 is turned on, channel A and channel B inputs must be cycled through their safe states at the same time before the circuit can be reset.

PLCR-020 SYS FAULT Robot Servos Enabled SS - Inputs Inconsistent

Cause:
The fault is generated if channel A and channel B are in inconsistent states (i.e., one safe and one active) for more than then 3000ms.

Remedy:
The fault indication is cleared when both channel A and channel B are in the safe state and the fault reset button is pressed.

PLCR-021 SYS FAULT Robot Servos Enabled SS - Fault Present

Cause:
This alarm is present whenever there is a cycle inputs or an inputs inconsistent alarm.

Remedy:
Fix the cycle inputs or inputs inconsistent alarm and press the fault reset button.
